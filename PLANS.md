# PLANS.md

このファイルは、このリポジトリの長めの開発ロードマップです。  
`AGENTS.md` は常設の判断基準、`PLANS.md` は multi-hour または複雑な作業で段階的に進めるときだけ使います。

小さな修正や単発タスクでは、このファイルに引っ張られすぎず、まず最小の動く変更を作ってください。

## ゴール

旅行の写真・動画から、流れが自然で見やすい動画を自動生成する。

最終的には次の順で進める。

1. `scene` 分割
2. 代表素材選定
3. 意味づけ
4. 編集構造の生成
5. LLM による構成案生成
6. レンダリング

## 現在地

今は次の段階まで到達している。

- 素材走査
- `scene` 分割
- 代表素材選定
- OCR / 顔検出を補助情報として付与
- 編集候補 JSON の出力

ここから先は、素材の認識精度を上げることが主目的ではない。  
動画としての見やすさを作るために、意味づけと編集構造を積み上げる。

## 開発方針

- 素材の意味理解を目的化しない
- OCR / 顔検出は補助として扱う
- 単体素材の正しさより、scene 全体の流れを優先する
- 編集に使える形へ情報を圧縮する
- LLM は最終的な構成生成に使い、前段はできるだけ安定した構造化データにする

## Phase 1: 意味づけ

scene ごとの内容を、編集で扱いやすい単位に要約する。

- `scene_summary`
- `scene_role`
- `importance_score`
- `importance_reason`
- `representative_tag`
- `representative_caption`

## Phase 2: 編集構造

scene を、動画の流れとして並べ替えられる構造にする。

- `edit_sequence`
- `segment_order`
- `transition_hint`
- `skip_candidate`
- `merge_candidate`

## Phase 3: LLM による構成生成

編集構造をもとに、LLM に全体構成を作らせる。

- `structure_outline`
- `chapter_list`
- `opening`
- `middle`
- `ending`
- `narration_plan`
- `subtitle_plan`

### 注意

- LLM は判断の最後段に寄せる
- LLM に生素材の細部を読ませすぎない
- 先に作った構造化データを中心に渡す

## Phase 4: レンダリング

構成案を実際の動画として書き出す。

- プレビュー動画
- 本番レンダリング
- レンダリング用ログ
- 失敗時の再実行しやすい中間成果物

## 優先順位

1. 意味づけを安定させる
2. 編集構造を作る
3. LLM で構成を生成する
4. レンダリングを自動化する

## 完了条件の目安

- scene ごとに何の場面かを説明できる
- その scene を動画で使う理由が説明できる
- scene の順序に意図がある
- LLM が構成案を作れるだけの入力がある
- レンダリングが再現可能に実行できる
